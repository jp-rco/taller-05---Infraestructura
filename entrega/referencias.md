# Referencias – Taller 5 (STRIDE)

1. Microsoft Learn – Threat Modeling Tool (amenazas y STRIDE): https://learn.microsoft.com/en-us/azure/security/develop/threat-modeling-tool-threats
2. OWASP – Threat Modeling Process (incluye STRIDE como mnemónico): https://owasp.org/www-community/Threat_Modeling_Process
3. OWASP – Application Security Verification Standard (ASVS): https://owasp.org/www-project-application-security-verification-standard/
4. OWASP – Top Ten Web Application Security Risks: https://owasp.org/www-project-top-ten/
5. NIST – SP 800-53 Rev. 5 (catálogo de controles de seguridad): https://csrc.nist.gov/pubs/sp/800/53/r5/upd1/final

Notas:
- Estas referencias se usaron como guía para controles típicos de autenticación, autorización, logging/auditoría, disponibilidad y buenas prácticas de hardening.
